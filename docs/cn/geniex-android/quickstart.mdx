有两种方式运行 GenieX Android 示例：安装预构建 APK（最快）或在 Android Studio 中从源码构建（可修改界面并锁定特定 SDK 版本）。

## **方式一：安装预构建 APK**

下载最新示例 APK：

<Card
  title="geniex-demo.apk"
  icon="download"
  href="https://qaihub-public-assets.s3.us-west-2.amazonaws.com/qai-hub-geniex/geniex-demo.apk"
>
  最新 GenieX Android 示例 APK。已签名的调试构建 — 通过 `adb install` 安装或上传到 QDC 会话。
</Card>

### **1a. 使用物理骁龙 8 Elite 设备**

需要主机上安装 [`adb`](https://developer.android.com/tools/adb)，以及一部 **骁龙 8 Elite**（SM8750）或 **骁龙 8 Elite Gen 5**（SM8850）手机。

<Steps>
  <Step title="启用 USB 调试">
    在手机上，打开 **设置 → 关于手机**，连续点击 **版本号** 七次，然后进入 **设置 → 系统 → 开发者选项**，启用 **USB 调试**。
  </Step>
  <Step title="连接并验证">
    ```bash
    adb devices
    ```
    手机应显示为 `device` 状态。如果显示 `unauthorized`，请在手机上接受 RSA 提示。
  </Step>
  <Step title="安装 APK">
    ```bash
    adb install -r geniex-demo.apk
    ```
    `-r` 允许覆盖安装之前的版本。
  </Step>
  <Step title="启动">
    从应用列表打开 **GenieX Demo**。首次使用时模型会从 Hugging Face / Qualcomm AI Hub 下载 — 请保持 Wi-Fi 连接。
  </Step>
</Steps>

### **1b. 无物理设备 — 使用高通设备云（QDC）**

如果没有骁龙 8 Elite 或 8 Elite Gen 5 手机，可以在高通设备云上按分钟租用。

<Steps>
  <Step title="选择设备">
    登录 [Qualcomm Device Cloud](https://qdc.qualcomm.com/)，选择 **Snapdragon 8 Elite** 或 **Snapdragon 8 Elite Gen 5**，然后选择 **Interactive session**。
  </Step>
  <Step title="配置会话">
    启动会话前，启用：
    - **Wi-Fi** — 示例应用需要从 Hugging Face / AI Hub 下载模型。
    - **保持屏幕常亮** — 避免下载过程中锁屏。

    此流程**不需要** SSH。
  </Step>
  <Step title="上传 APK">
    **在启动会话之前**，在会话设置页面使用 **Upload file** 将 `geniex-demo.apk` 推送到设备。文件必须在会话开始前上传。
  </Step>
  <Step title="启动并测试">
    启动会话。在镜像屏幕中点击已上传的 APK 进行安装，然后从应用列表打开 **GenieX Demo** — 无需本地 adb。
  </Step>
  <Step title="（可选）使用图片测试 VLM">
    要试用视觉语言模型，可通过 QDC 终端上传测试图片：
    ```bash
    curl -L "https://s7d1.scene7.com/is/image/dmqualcommprod/Qualcomm_AIHub_image2-1?$QC_Responsive$&fmt=png-alpha" -o /data/local/tmp/qualcomm.png
    cp /data/local/tmp/qualcomm.png /sdcard/Download/
    ```
    图片会出现在 VLM 聊天中点击图像按钮后的图片选择器中。
  </Step>
</Steps>

## **方式二：从源码构建示例应用**

在 Android Studio 中构建示例 — 适用于修改界面或锁定特定已发布的 SDK 版本。

<Note>
Gradle 从 **Maven Central** 拉取 GenieX SDK（坐标：`com.qualcomm.qti:geniex-android:0.1.6`）。此方式**不需要**自行构建原生 SDK。
</Note>

<Steps>
  <Step title="克隆仓库">
    ```bash
    git clone https://github.com/qcom-ai-hub/geniex.git
    cd geniex
    ```
  </Step>
  <Step title="在 Android Studio 中打开示例项目">
    打开 **`examples/android/`**（示例应用）。**不要**打开 `bindings/android/` — 那是 SDK 绑定模块，此流程通过 Maven Central 消费。
  </Step>
  <Step title="同步 Gradle">
    Android Studio 会提示同步。同步会从 Maven Central 解析 `com.qualcomm.qti:geniex-android:0.1.6` — 这是示例需要的唯一 SDK 依赖。
  </Step>
  <Step title="连接设备">
    将启用了 USB 调试的骁龙 8 Elite 或 8 Elite Gen 5 手机插入电脑（参见方式 1a），让 Android Studio 检测到它。
  </Step>
  <Step title="运行">
    点击 **Run ▶**。Android Studio 构建 APK，安装到手机并启动示例。
  </Step>
</Steps>

## **使用示例应用**

应用启动后：

1. **选择模型** — 从顶部下拉菜单选择：
   - **Hugging Face GGUF (llama.cpp)** — Qwen3、Phi-4、Gemma、Granite、Qwen3-VL 等。
   - **AI Hub NPU (QAIRT)** — `qualcomm/Qwen3-4B-Instruct-2507`、`qualcomm/Qwen2.5-VL-7B-Instruct`。
2. **下载** — 点击 Download。进度可断点续传：关闭应用后重启会从断点继续。
3. **加载** — 弹窗让你选择执行目标（GGUF 可选 NPU / GPU / CPU；AI Hub 模型仅 NPU）。
4. **聊天** — 输入文字，或点击图像按钮使用 VLM。
