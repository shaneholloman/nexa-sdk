## **概述**

GenieX 内置本地推理服务器，提供 **OpenAI 兼容 API**。
你可以在设备端运行模型，并将其连接到支持 OpenAI 协议的应用或框架
——包括 **LangChain** 等智能体工作流、**OpenClaw** 等 AI 原生应用以及你自己的自定义应用。
无需云端依赖——仅需一个由高通硬件加速支持的本地端点。

## **快速开始**

如果尚未拉取模型，请先下载。例如，拉取 Qwen3：

```bash bash
geniex pull qualcomm/Qwen3-4B-Instruct-2507 
```

使用 API 前，先从项目根目录打开终端，然后启动 GenieX 服务器：

```bash bash
geniex serve
```

服务器默认运行在 `http://127.0.0.1:18181`。\
保持运行服务器的终端窗口打开，在另一个终端标签页中发送请求。\
查看服务器所有可配置选项，运行 `geniex serve -h`


## **/v1/chat/completions**

为给定对话创建模型响应。支持 LLM（纯文本）和 VLM（图像+文本）。

### **使用 LLM**

#### **请求体**

```json Example Value
{
  "model": "qualcomm/Qwen3-4B-Instruct-2507",
  "messages": [
    {"role": "user", "content": "Hello! Briefly introduce yourself."}
  ],
  "max_tokens": 256,
  "temperature": 0.7,
  "stream": false
}
```

#### **使用示例**

在浏览器中打开 `http://127.0.0.1:18181` 即可访问内置的 Swagger UI。

**步骤 1.** 展开 `POST /v1/chat/completions` 端点，查看示例请求体和 schema。

![Swagger UI 展示 chat completions 端点及示例请求体](/Images/server/curl-1.png)

**步骤 2.** 点击 **Try it out**，根据需要编辑请求体，然后点击 **Execute**。

![在 Try-it-out 模式下编辑请求体后执行](/Images/server/curl-2.png)

**步骤 3.** 查看响应——`200` 状态码以及模型生成的回复。

![响应体展示 200 成功响应及模型回复](/Images/server/curl-3.png)


### **使用 VLM**

#### **请求体**

`image_url` 可以是以下任一格式：

- 服务器本地文件路径（需服务器有访问权限），例如 `C:/Users/Username/Pictures/photo.jpg`。`file://` 前缀可选。

首先，下载一张示例图片到当前目录：

```bash
curl -o demo.jpg https://qaihub-public-assets.s3.us-west-2.amazonaws.com/qai-hub-geniex/demo.jpg
```

```json Example Value
{
  "model": "qualcomm/Qwen2.5-VL-7B-Instruct",
  "messages": [
    {
      "role": "user",
      "content": [
        {"type": "text", "text": "Describe this image succinctly."},
        {"type": "image_url", "image_url": {"url": "</path/to/image>"}}
      ]
    }
  ]
}
```

#### **使用示例**

在浏览器中打开 `http://127.0.0.1:18181` 即可访问内置的 Swagger UI。

**步骤 1.** 展开 `POST /v1/chat/completions` 端点，查看示例请求体和 schema。

![Swagger UI 展示 chat completions 端点及示例请求体](/Images/server/curl-1.png)

**步骤 2.** 点击 **Try it out**，将请求体替换为 VLM payload（`image_url.url` 指向本地图片路径），然后点击 **Execute**。

![在 Try-it-out 模式下编辑 VLM 请求体后执行](/Images/server/curl-4.png)

**步骤 3.** 查看响应——`200` 状态码以及模型对图像的描述。

![响应体展示 200 成功响应及 VLM 对图像的描述](/Images/server/curl-5.png)
