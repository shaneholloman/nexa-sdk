## **安装**

### **拉取 Docker 镜像**

使用你的 GitHub PAT（个人访问令牌）和 GitHub 用户名登录 GitHub Container Registry (GHCR)：

```bash bash
export GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxxxx
export GITHUB_USERNAME=your_github_username
docker login ghcr.io -u "$GITHUB_USERNAME" -p "$GITHUB_TOKEN"
```

拉取最新 GenieX SDK Docker 镜像：

```bash bash
docker pull ghcr.io/qcom-ai-hub/geniex-cli:latest
```

## **使用模式**

GenieX SDK Docker 支持两种使用模式：服务器模式用于 REST API 访问，交互式 CLI 模式用于直接模型推理。


### **CLI 模式（交互式）**

以交互式 CLI 模式运行 GenieX SDK 进行直接模型推理：

```bash bash
docker run -it --rm --privileged \
  -v "$PWD/data:/data" \
  -v /usr/lib:/opt/qcom-lib:ro \
  ghcr.io/qcom-ai-hub/geniex-cli:latest \
  infer Qwen/Qwen3-0.6B-GGUF
```

<Note>
`-it` 标志启用交互模式。将 `Qwen/Qwen3-0.6B-GGUF` 替换为任意支持的模型名称。完整的支持模型列表请查看[概览](/cn/geniex-docker/overview#支持的模型)页面。
</Note>


### **服务器模式（后台运行）**

以服务器模式运行 GenieX SDK 以暴露 REST API 端点。此模式在后台运行：

```bash bash
docker run -it --rm --privileged \
  -v "$PWD/data:/data" \
  -v /usr/lib:/opt/qcom-lib:ro \
  --network=host \
  ghcr.io/qcom-ai-hub/geniex-cli:latest \
  serve
```

<Note>
`--privileged` 标志是 NPU/GPU 设备访问所必需的。`--network=host` 将服务器暴露在主机网络上，可通过 `http://localhost:18181` 访问。
</Note>

`geniex serve` 不会自动下载模型。请确保预先下载要使用的模型。

服务器将在 `http://localhost:18181` 上可访问。详细 API 文档请查看 [REST API](/cn/geniex-docker/RestAPI) 页面。
