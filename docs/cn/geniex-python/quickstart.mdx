## **前置条件**

GenieX Python SDK 需要 ARM64 版本的 Python。在安装之前，请下载并安装 [Python 3.13.3 for ARM64](https://www.python.org/ftp/python/3.13.3/python-3.13.3-arm64.exe)。

## **方法 1：使用 Python 脚本**

对于 QDC，我们推荐使用以下 Python 脚本方法。

### **安装**

在 QDC 上，创建虚拟环境并使用 pip 安装 GenieX Python SDK：

```powershell
python -m venv geniex-env
.\geniex-env\Scripts\Activate.ps1
pip install -i https://test.pypi.org/simple/ --extra-index-url https://pypi.org/simple geniex
```
这将从 [test PyPI](https://test.pypi.org/project/geniex/) 下载 Python 包。

<Note>
**故障排查：SSL / 证书错误（企业代理 / QDC）**

如果安装时出现 SSL 证书错误——常见于启用了 TLS 检测的企业网络，包括 **Qualcomm Developer Cloud（QDC）**——请在运行 `pip install` 之前，按照以下 **PowerShell** 命令手动下载 SDK：

```powershell
$env:GENIEX_VERSION = "v0.1.4"  # 示例版本号，请替换为您实际安装的版本
$base = "https://qaihub-public-assets.s3.us-west-2.amazonaws.com/qai-hub-geniex/$env:GENIEX_VERSION"
$zip  = "geniex-sdk-windows-arm64-$env:GENIEX_VERSION.zip"

Invoke-WebRequest "$base/$zip"        -OutFile $zip
Invoke-WebRequest "$base/$zip.sha256" -OutFile "$zip.sha256"

$env:GENIEX_SDK_DOWNLOAD_URL = "file:///$($(Get-Location).Path.Replace('\','/'))"
pip install -i https://test.pypi.org/simple/ --extra-index-url https://pypi.org/simple geniex

Remove-Item $zip, "$zip.sha256"
Remove-Item Env:GENIEX_SDK_DOWNLOAD_URL
```
</Note>

### **LLM 推理**

以下是一个简单的入门示例：

```python
from geniex import AutoModelForCausalLM

# GGUF 模型示例
model = AutoModelForCausalLM.from_pretrained(
    "Qwen/Qwen3-0.6B-GGUF",     # GGUF 模型的 HF 仓库 ID，或本地 .gguf LLM 模型路径
    device_map="auto",          # "auto"（默认）| "cpu" | "gpu" | "npu" | "hybrid"
                                # | "<plugin>" | "<plugin>:<device>"
                                # auto -> llama_cpp 时为 hybrid（HTP+CPU 快速路径），qairt 时为 npu
)

# QAIRT 模型示例
# model = AutoModelForCausalLM.from_pretrained(
#     r"C:\<your_path>\Qwen3-4B-Instruct-2507\",    # 替换为您的本地路径；传入模型文件夹中任意一个 .bin 文件的路径。
#     model_name="qwen3_4b_instruct_2507",          # 模型名称与 QAI Hub Model 上使用的 model id 相同；可在 S3 下载链接中查找。
#     device_map="qairt",                           # QAIRT 模型必须使用 device_map="qairt"
# )


messages = [{"role": "user", "content": "What is 2+2?"}]
prompt = model.tokenizer.apply_chat_template(
    messages, add_generation_prompt=True,
)

# 一次性生成
output = model.generate(prompt, max_new_tokens=256)
print(output.text)
print(f"[{output.profile.generated_tokens} tok, "
      f"{output.profile.decode_speed:.1f} tok/s, stop={output.profile.stop_reason}]")

# 流式生成
streamer = model.generate(prompt, max_new_tokens=256, stream=True)
for chunk in streamer:
    print(chunk, end="", flush=True)

model.close()
```

### **VLM 推理**

首先，下载一张示例图片：

```bash
curl -o demo.jpg https://qaihub-public-assets.s3.us-west-2.amazonaws.com/qai-hub-geniex/demo.jpg
```

然后使用下载图片的完整路径运行推理：

```python
import os
from geniex import AutoModelForVision2Seq

image_path = os.path.abspath("demo.jpg")

model = AutoModelForVision2Seq.from_pretrained("unsloth/Qwen2.5-VL-3B-Instruct-GGUF", quant="Q8_0", device_map="auto")
messages = [{
    "role": "user",
    "content": [
        {"type": "image", "image": image_path},
        {"type": "text", "text": "Describe the image."},
    ],
}]
prompt = model.tokenizer.apply_chat_template(
    messages, tokenize=False, add_generation_prompt=True,
)
print(model.generate(prompt, images=[image_path]).text)
```
