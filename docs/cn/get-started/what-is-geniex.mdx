GenieX 是为 Qualcomm 平台打造的端侧生成式 AI 推理运行时。它是在 Hexagon NPU、Adreno GPU 或 CPU 上用几行代码本地运行前沿语言模型与视觉语言模型的最简单方式。它是 Qualcomm GENIE 的社区版。

## **架构**

GenieX 通过同一套 SDK 暴露五种入口——CLI、Python、Java/Kotlin、Docker 以及兼容 OpenAI 协议的服务器，并由该 SDK 按需分发到 **llama.cpp 运行时**（覆盖 CPU / GPU / Hexagon HTP 的 GGML 内核）或 **[Qualcomm® AI Engine Direct](https://www.qualcomm.com/developer/software/qualcomm-ai-engine-direct-sdk) 运行时**（仅 NPU）。同一套 SDK 可运行在 Windows ARM64、Android 与 Linux ARM64 上。

<Note>**Qualcomm AI Engine Direct** 是官方名称，它也被称为 *Qualcomm AI Engine Direct SDK*、*Qualcomm AI Runtime*，以及历史上的 *QAIRT*。本文档中我们统一使用该官方名称。</Note>

## **为什么是两种运行时？**

这样你就能在同一套 SDK 中同时获得**广泛的模型覆盖**与**最佳性能**：

- **大多数模型开箱即用** —— 将 GenieX 指向 Hugging Face 上几乎任意 GGUF 模型，它即可通过 llama.cpp 在 CPU / GPU / NPU 上运行。
- **Qualcomm® AI Hub 模型以最佳方式运行** —— 发布到 [Qualcomm AI Hub](https://aihub.qualcomm.com/) 的模型已按芯片预编译，并通过 Qualcomm AI Engine Direct 在 Hexagon NPU 上运行，以获得端侧的峰值性能。

何时选哪一种，详见[平台与运行时](/cn/get-started/platforms#geniex-运行时)。

## **GenieX 能做什么**

- **本地运行模型** —— 覆盖骁龙 X（Windows ARM64）、骁龙 8 至尊版（Android）以及 Dragonwing IoT 芯片。
- **选择运行时** —— `llama.cpp` 适配任意社区 GGUF 模型，Qualcomm AI Engine Direct（`qairt`）适配 Qualcomm AI Hub 预编译的 NPU 模型包。
- **构建应用** —— 通过 CLI、兼容 OpenAI 协议的本地服务器、Python SDK、Android SDK 或 Docker 镜像。
