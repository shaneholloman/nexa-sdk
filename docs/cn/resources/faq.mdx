## **通用**

<AccordionGroup>
  <Accordion title="GenieX 是什么？">
    面向高通骁龙的多平台 AI 推理 SDK。GenieX 在 Hexagon NPU、Adreno GPU 或 CPU 计算单元上端侧运行前沿 LLM 与 VLM，覆盖 Windows ARM64、Android 与 Linux ARM64。详见 [GenieX 是什么](/cn/get-started/what-is-geniex)。
  </Accordion>

  <Accordion title="应该选哪种接入方式？">
    - **试用、写脚本** —— Windows ARM64 或 Linux ARM64 上的 [CLI](/cn/run/cli/install)。
    - **构建应用** —— [本地服务器](/cn/run/cli/local-server)（兼容 OpenAI 协议的 HTTP）或 [Python SDK](/cn/run/python/install)。
    - **移动端** —— [Android SDK](/cn/run/android/install)（Kotlin，Maven Central）。
    - **可复现的 IoT 部署** —— Linux ARM64 EVK 上的 [Docker 镜像](/cn/run/cli/install)，锁定到指定 release tag。

    [快速入门](/cn/get-started/quickstart)是这四条路径的统一入口。
  </Accordion>

  <Accordion title="哪里能看到 GenieX 支持的硬件？">
    [支持的平台](/cn/get-started/platforms) —— Windows ARM64 上的 Snapdragon X、Android 上的骁龙 8 至尊版、Linux ARM64 上的 Dragonwing QCS9075。
  </Accordion>
</AccordionGroup>

## **运行时**

<AccordionGroup>
  <Accordion title="llama.cpp 与 Qualcomm AI Engine Direct 有什么区别？">
    - **llama.cpp** —— 运行任意 GGUF 模型，支持 NPU/GPU/CPU 计算单元。最适合试用社区模型。
    - **[Qualcomm® AI Engine Direct](https://www.qualcomm.com/developer/software/qualcomm-ai-engine-direct-sdk)**（`qairt`）—— 运行 Qualcomm AI Hub 预编译模型，仅 NPU。当模型在 Qualcomm AI Hub 可用时，通常是最快路径。

    详见[平台与运行时](/cn/get-started/platforms#geniex-运行时)。
  </Accordion>

  <Accordion title="默认运行时是哪个？">
    如果不传计算单元：
    - `llama_cpp` 默认 `hybrid`（HTP + CPU 按张量调度——骁龙上的快速路径）。
    - `qairt` 默认 `npu`。

    部分模型会覆盖默认值——例如 `gpt-oss` 在 `llama_cpp` 下回退到 `npu`。
  </Accordion>

  <Accordion title="应该选哪种精度（量化）？">
    llama.cpp 在骁龙 NPU 上请用 **`Q4_0`**——它在 Hexagon NPU 上支持最佳。Qualcomm AI Hub 模型已预量化——无需选择。详见[精度（量化）与设备落点](/cn/models/supported#精度量化与设备落点)。
  </Accordion>
</AccordionGroup>

## **模型**

<AccordionGroup>
  <Accordion title="能运行 Hugging Face 上任意模型吗？">
    llama.cpp 运行时可执行任意 **GGUF** 模型。详见[运行 Hugging Face 上的 GGUF 模型](/cn/models/supported#运行-hugging-face-上的-gguf-模型)。

    Qualcomm AI Engine Direct 需要预编译的 Qualcomm AI Hub 模型——新增模型需先在 C++ 侧注册。
  </Accordion>

  <Accordion title="受支持模型列表在哪里？">
    [模型](/cn/models/supported) —— 按运行时拆分（Qualcomm AI Engine Direct、llama.cpp）。
  </Accordion>
</AccordionGroup>

## **芯片与设备**

<AccordionGroup>
  <Accordion title="必须用骁龙设备吗？">
    是。GenieX 面向高通骁龙芯片——Hexagon NPU、Adreno GPU 与骁龙 ARM CPU 计算单元。没有 x86 或非骁龙 ARM 构建。如手头无设备，可使用 [Qualcomm Developer Cloud / Device Cloud](/cn/get-started/platforms)。
  </Accordion>

  <Accordion title="我的 Qualcomm AI Hub 模型提示需要 NPU——能在 CPU 上跑吗？">
    不能。Qualcomm AI Engine Direct 设计上仅支持 NPU。`cpu` 与 `gpu` 别名会被自动转为 NPU 并打印告警。如需 CPU/GPU 回退，请改用 llama.cpp 运行时（GGUF 模型）。
  </Accordion>
</AccordionGroup>

## **QDC（Qualcomm Device Cloud）**

<AccordionGroup>
  <Accordion title="我必须有自己的 QDC 设备吗？">
    不需要。登录 [Qualcomm Developer Cloud](https://qdc.qualcomm.com/)，选择一台骁龙设备，启动 Interactive Session——整个流程就是这样。哪些芯片在支持范围内，详见[支持的平台](/cn/get-started/platforms)。
  </Accordion>

  <Accordion title="如何通过 SSH 连接到 QDC 设备？">
    <Steps>
      <Step title="建立 SSH 隧道">
        在 QDC 控制台中前往 **Interactive Sessions**，点击 **Connect** 查看 SSH 连接说明。

        <img
          src="/Images/qdc/qdc-1.png"
          alt="QDC portal showing the Connect button for SSH instructions"
          style={{ borderRadius: '0.5rem' }}
        />

        打开终端，运行 QDC 提供的 SSH 隧道命令，把 `<PRIVATE_KEY_FILE_PATH>` 替换为你的私钥路径。

        <img
          src="/Images/qdc/qdc-2.png"
          alt="SSH tunneling setup with private key and port forwarding"
          style={{ borderRadius: '0.5rem' }}
        />

        <Tip>
        **`WARNING: UNPROTECTED PRIVATE KEY FILE!`** —— 你的 `.pem` 文件权限过宽。SSH 要求私钥仅文件所有者可读：

        ```bash
        chmod 600 /path/to/your-key.pem
        ```
        </Tip>
      </Step>

      <Step title="连接到设备">
        在新终端中通过转发端口 SSH 进入设备。默认密码为 `oelinux123`：

        ```bash
        ssh -o StrictHostKeychecking=no -o UserKnownHostsFile=/dev/null -p 2222 root@localhost
        ```

        <img
          src="/Images/qdc/qdc-3.png"
          alt="Successful SSH connection to the QDC device"
          style={{ borderRadius: '0.5rem' }}
        />
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="没有实体设备如何测试 Android demo？" id="how-do-i-test-the-android-demo-without-a-physical-device">
    在 QDC 上使用 **Snapdragon 8 Elite** 或 **8 Elite Gen 5** 的 Interactive Session，并安装预构建的 demo APK。

    <Card title="geniex-demo.apk" icon="download" href="https://qaihub-public-assets.s3.us-west-2.amazonaws.com/qai-hub-geniex/geniex-demo.apk">
      最新的 GenieX Android demo APK。已签名的 debug 构建。
    </Card>

    <Steps>
      <Step title="选择设备">
        登录 [Qualcomm Device Cloud](https://qdc.qualcomm.com/)，选择 **Snapdragon 8 Elite** 或 **Snapdragon 8 Elite Gen 5**，并选择 **Interactive session**。
      </Step>
      <Step title="配置会话">
        启用 **Wi-Fi**（让 demo 能下载模型）和 **Keep screen on**（避免下载中途锁屏）。**无需** SSH。
      </Step>
      <Step title="上传 APK">
        在会话设置页用 **Upload file** 把 `geniex-demo.apk` 推送到设备——该文件必须在会话开始**之前**上传。
      </Step>
      <Step title="启动并测试">
        启动会话。在镜像屏幕中点击已上传的 APK 进行安装，然后从应用抽屉打开 **GenieX Demo**。
      </Step>
      <Step title="（可选）用图片测试 VLM">
        通过 QDC 终端上传一张测试图片：
        ```bash
        curl -L "https://s7d1.scene7.com/is/image/dmqualcommprod/Qualcomm_AIHub_image2-1?$QC_Responsive$&fmt=png-alpha" -o /data/local/tmp/qualcomm.png
        cp /data/local/tmp/qualcomm.png /sdcard/Download/
        ```
        该图片会出现在应用的图片选择器中。
      </Step>
    </Steps>
  </Accordion>
</AccordionGroup>

## **服务器 / API**

<AccordionGroup>
  <Accordion title="本地服务器兼容 OpenAI 吗？">
    是——详见[本地服务器](/cn/run/cli/local-server)。可将官方 `openai` Python 客户端指向 `http://127.0.0.1:18181/v1`，OpenAI 代码无需修改即可复用。
  </Accordion>

  <Accordion title="geniex serve 会自动下载模型吗？">
    不会。请先用 `geniex pull` 拉取模型再启动服务器。
  </Accordion>
</AccordionGroup>

## **Linux ARM64 环境**

<AccordionGroup>
  <Accordion title="check.sh 报告缺少库——我该安装什么？">
    预检脚本 (`check.sh`) 与 Docker 入口脚本 (`entrypoint.sh`) 会提示系统库或高通驱动缺失。请在**宿主机**上运行以下命令——如果当前在 geniex 容器内，请先 `exit`。在 Ubuntu / Debian 上用两条 `apt` 命令安装。

    **1. 基础系统库（来自 Ubuntu 官方源）：**

    ```bash
    sudo apt update
    sudo apt install -y \
      libatomic1 \
      libglib2.0-0 \
      ocl-icd-libopencl1
    ```

    **2. 高通驱动包（Adreno GPU + FastRPC NPU 桥）：**

    ```bash
    sudo apt-get install -y qcom-adreno1 qcom-fastrpc1
    ```

    高通驱动包来自厂商 BSP / EVK 仓库——不在 Ubuntu 官方镜像中。安装完成后，在裸机上重新运行 `check.sh`，或重启容器来验证；若裸机上 `libcdsprpc.so` / `libadsprpc.so` 软链接仍缺失，请重新运行 geniex 安装脚本 (`install.sh`) 重建。
  </Accordion>
</AccordionGroup>
