## **模型推理**

### **`geniex pull`**

下载模型并存储到本地。

```powershell
geniex pull <model-name>
```

CLI 会引导你完成设置流程：

**1. 设备选择**

首次运行时（QAIRT 与 llama.cpp 插件均适用），系统会提示选择设备：

```powershell
Select your device
┃ > Snapdragon X Elite CRD
┃   Snapdragon X Plus 8-Core CRD
┃   Snapdragon X2 Elite CRD
```

如需更改：

```powershell
geniex config set device
```

**2. 量化（仅 llama.cpp）**

对于 GGUF 模型，CLI 会提示选择量化：

```powershell
Choose a quant version to download
> Q4_0       [1.2 GiB] (default)
  Q8_0       [2.0 GiB]
  F16        [3.8 GiB]
```

<Tip>`Q4_0` 在 Hexagon NPU 上支持最佳。详见[量化](/cn/models/quantization)。</Tip>

QAIRT 模型已预量化——无需选择。

### **`geniex infer` — LLM**

启动与语言模型的交互式对话。

```powershell
geniex infer ai-hub-models/Qwen3-4B
```

**思考模式** —— 控制模型是否在回复前展示推理过程：

```powershell
geniex infer ai-hub-models/Qwen3-4B --think         # 展示推理步骤
geniex infer ai-hub-models/Qwen3-4B --think=false   # 直接回复
```

**计算单元选择**（通过 `--device`）—— 选择运行模型的计算单元（默认：`npu`）：

```powershell
# llama.cpp 模型支持所有计算单元
geniex infer unsloth/Qwen3.5-0.8B-GGUF --device npu
geniex infer unsloth/Qwen3.5-0.8B-GGUF --device gpu
geniex infer unsloth/Qwen3.5-0.8B-GGUF --device cpu

# QAIRT 模型仅支持 NPU
geniex infer ai-hub-models/Qwen3-4B --device npu
```

<Warning>QAIRT 模型仅在 NPU 上运行。对 QAIRT 模型使用 `--device cpu` 或 `--device gpu` 会返回错误。</Warning>

### **`geniex infer` — VLM**

运行视觉语言模型推理，支持纯文本或图像输入：

```bash
geniex infer ai-hub-models/Qwen2.5-VL-7B-Instruct-GGUF
```

如果只需文本输入，直接启动并对话即可。若要让模型基于图像回复，请提供**绝对路径**或将图片文件直接拖入终端：

```bash
Describe this picture </full/path/to/image.png>
```

## **`geniex serve`**

启动兼容 OpenAI 协议的本地服务器。API 详见[本地服务器](/cn/run/cli/local-server)。

```bash
geniex serve
```

## **配置标志**

以下标志可传给 `geniex infer`，用于控制模型加载与生成行为。

### 采样器标志

控制模型在生成时如何选择 token。

| 标志 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `--temperature` | float | — | 采样温度。值越高随机性越大。 |
| `--top-p` | float | — | Top-p（核采样）阈值。 |
| `--top-k` | int | — | Top-k 采样。仅考虑概率最高的 k 个 token。 |
| `--min-p` | float | — | Min-p 采样阈值。 |
| `--repetition-penalty` | float | `1` | 重复惩罚。值 > 1 减少重复。 |
| `--presence-penalty` | float | — | 对已出现的 token 进行惩罚。 |
| `--frequency-penalty` | float | — | 按频率比例惩罚 token。 |
| `--seed` | int | — | 随机种子，用于可复现输出。 |
| `--grammar-path` | string | — | GBNF 语法文件路径，用于约束生成。 |
| `--grammar-string` | string | — | 内联 GBNF 格式语法字符串。 |
| `--enable-json` | — | — | 强制仅输出 JSON。 |

### 模型标志

控制模型加载、上下文与生成限制。

| 标志 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `-n`, `--ngl` | int | `999` | 卸载到 GPU 的层数。 |
| `--nctx` | int | `4096` | 上下文窗口大小（最大输入 + 输出 token 数）。 |
| `--max-tokens` | int | `2048` | 每次响应生成的最大 token 数。 |
| `--stop` | string[] | — | 停止序列（可多次指定）。 |
| `--stop-file` | string | — | 包含停止序列的文件（每行一个）。 |
| `--think` / `--think=false` | bool | `true` | 启用或禁用推理模型的思考模式。 |
| `-s`, `--system-prompt` | string | — | 设置模型行为的系统提示。 |

## **实用命令**

| 命令 | 说明 | 示例 |
|---|---|---|
| `geniex list` | 显示所有已下载模型的名称与大小。 | `geniex list` |
| `geniex remove <model>` | 按名称删除指定的本地模型。 | `geniex remove unsloth/Qwen3-0.6B-GGUF` |
| `geniex clean` | 删除所有本地缓存的模型。 | `geniex clean` |
| `geniex infer -h` | 显示 `geniex infer` 的帮助菜单。 | `geniex infer -h` |
| `geniex serve -h` | 显示 `geniex serve` 的帮助菜单。 | `geniex serve -h` |
