Linux Docker 路径通过 Linux ARM64 容器镜像运行，并支持 NPU 访问（适用于 Dragonwing IQ9 等 IoT 平台）。

## **CLI**

完整 CLI Docker 安装指南，参见 [CLI 安装 — Linux ARM64 (Docker)](/cn/run/cli/install)。

### **前置条件**

- 安装了 Docker 的 Linux ARM64 主机。
- 高通 IoT 芯片——详见[支持的平台](/cn/get-started/platforms#iot-linux-arm64)。
- 拥有 GHCR 读权限的 GitHub Personal Access Token (PAT)。

### **拉取镜像**

登录 GitHub Container Registry 后再拉取：

```bash bash
export GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxxxx
docker login ghcr.io -u <your-github-username> -p "$GITHUB_TOKEN"

docker pull ghcr.io/qcom-ai-hub/geniex-cli:latest
```

<Note>
  若 `docker login` 失败或 `docker pull` 返回 `permission denied ... /var/run/docker.sock`，参见[故障排查 → Linux](/cn/resources/troubleshooting#linux) 了解 PAT 权限与 `docker` 组的修复方法。
</Note>

### **交互式运行**

进入容器内交互式 shell：

```bash bash
docker run -it --rm --privileged \
  -v "$PWD/data:/data" \
  -v /usr/lib:/opt/qcom-lib:ro \
  ghcr.io/qcom-ai-hub/geniex-cli:latest
```

### **验证**

在容器 shell 中运行测试命令：

```bash bash
geniex --help
```

<Note>NPU 访问需要 `--privileged` 标志。</Note>

服务器模式参见[本地服务器](/cn/run/cli/local-server)。
