This page covers the configuration knobs that apply to every wrapper: which backend to load, which device to run on, and the data types used to create wrappers and stream results. For end-to-end examples, see [CPU / GPU / Hybrid](/en/geniex-android/APIReference_CPUGPU) and [NPU](/en/geniex-android/APIReference_NPU).

## **Plugin selection**

Choose the inference backend via `plugin_id`:

```kotlin
val plugin_id: String?   // "llama_cpp" | "qairt" | null
```

| `plugin_id`    | Backend                                                                                       | Model format             | Devices                          |
|----------------|-----------------------------------------------------------------------------------------------|--------------------------|----------------------------------|
| `"llama_cpp"`  | llama.cpp + GGML Hexagon backend                                                              | GGUF                     | CPU / Adreno GPU / Hexagon NPU   |
| `"qairt"`      | Qualcomm AI Runtime with QAIRT pipelines                                                      | AI Hub pre-compiled bins | Hexagon NPU only                 |
| `null`         | Let the SDK pick based on the resolved model paths                                            | —                        | —                                |

The string constants are also exposed as [`PluginIdValue`](https://github.com/qcom-ai-hub/geniex/blob/main/bindings/android/app/src/main/java/com/geniex/sdk/bean/InputPluginBase.kt) (`LLAMA_CPP`, `QAIRT`).

<Note>
The old `"cpu_gpu"` / `"npu"` / `"whisper_cpp"` / `"tts_cpp"` plugin ids from earlier documentation are no longer in use. llama.cpp now covers CPU / GPU / Hexagon NPU through `device_id`; QAIRT replaces the old NEXA NPU plugin.
</Note>

## **Device selection**

Friendly hardware aliases. The JNI layer forwards these to `geniex_resolve_device` in the native SDK — the single source of truth for `device_id` + `nGpuLayers` mapping.

```kotlin
val device_id: String?   // "cpu" | "gpu" | "npu" | "hybrid" | null
```

| Alias      | What it does                                                                                     |
|------------|--------------------------------------------------------------------------------------------------|
| `null`     | Plugin default — `hybrid` for `llama_cpp`, `npu` for `qairt`.                                    |
| `"cpu"`    | Pure CPU. JNI forces `nGpuLayers = 0`.                                                            |
| `"gpu"`    | Adreno GPU via OpenCL (`llama_cpp` only).                                                         |
| `"npu"`    | Pinned Hexagon session (`HTP0`). Deterministic; slower than `hybrid` on most LLMs.                |
| `"hybrid"` | `llama_cpp` per-tensor HTP+CPU scheduler. JNI forces `nGpuLayers = 999`. Fastest LLM path.        |

Also exposed as [`DeviceIdValue`](https://github.com/qcom-ai-hub/geniex/blob/main/bindings/android/app/src/main/java/com/geniex/sdk/bean/InputPluginBase.kt).

<Note>
QAIRT has only an NPU device. Passing `"cpu"`, `"gpu"`, or `"hybrid"` with a QAIRT model logs a warning and falls back to NPU — it will not fail the call.
</Note>

<Note>
Some model families can't run on `hybrid`. For example, `gpt-oss-*` is pinned to `npu` automatically via `is_llama_cpp_hybrid_incompatible` in the native SDK (`sdk/src/device.cpp`). Callers do not need to special-case this.
</Note>

## **Model manager**

Models are pulled on-device through the bundled Rust model manager. Do **not** manually `adb push` weights into `filesDir/models/` — that path is legacy and will be wiped on next launch.

### `ModelManagerWrapper`

Singleton `object` exposed at `com.geniex.sdk.ModelManagerWrapper`.

```kotlin
// One-time init. Safe to call on every Activity.onCreate — idempotent.
GeniexSdk.getInstance().init(context)   // calls ModelManagerWrapper.init() for you

// Pull, streaming progress.
ModelManagerWrapper.pullFlow(
    ModelPullInput(
        model_name = "unsloth/Qwen3-0.6B-GGUF",
        quant      = "Q4_0",
        hub        = HubSource.HUGGINGFACE,
    )
).collect { event ->
    when (event) {
        is ModelManagerWrapper.PullEvent.Progress  -> /* update UI */
        ModelManagerWrapper.PullEvent.Completed    -> /* done */
        is ModelManagerWrapper.PullEvent.Error     -> /* show error */
    }
}

// Resolve on-disk paths for a previously-pulled model.
val paths: ModelPaths? = ModelManagerWrapper.getPaths("unsloth/Qwen3-0.6B-GGUF")

// Inventory / cleanup.
ModelManagerWrapper.list()                  // List<String> of resolved model names
ModelManagerWrapper.remove("org/repo")      // Int (0 = success)
ModelManagerWrapper.clean()                 // wipe all cached models
```

### `ModelPullInput`

```kotlin
data class ModelPullInput(
    val model_name:   String,                     // "org/repo" or alias
    val quant:        String? = null,             // e.g. "Q4_0", "Q4_K_M"
    val hub:          HubSource = HubSource.AUTO, // AUTO routes by model_name
    val local_path:   String? = null,             // only when hub == LOCALFS
    val hf_token:     String? = null,             // falls back to GENIEX_HFTOKEN env
    val chipset:      String? = null,             // required for AI Hub on Android (e.g. "SM8750")
    val display_name: String? = null,             // required when hub == AIHUB explicitly
)
```

### `HubSource`

```kotlin
enum class HubSource(val value: Int) {
    AUTO(0),          // routes by model_name prefix (e.g. qualcomm/* → AIHUB)
    HUGGINGFACE(1),
    MODELSCOPE(2),
    AIHUB(3),
    VOLCES(4),
    LOCALFS(127),
}
```

### `ModelPaths`

Returned by `getPaths()`. Feed the fields straight into `*CreateInput`:

```kotlin
data class ModelPaths(
    val model_path:     String,
    val model_dir:      String,
    val model_name:     String,
    val plugin_id:      String,            // authoritative — prefer this over UI selection
    val mmproj_path:    String? = null,    // VLM projection weights (llama.cpp)
    val tokenizer_path: String? = null,
    val device_id:      String? = null,
)
```

### `FileProgress`

```kotlin
data class FileProgress(
    val file_name:        String,
    val downloaded_bytes: Long,
    val total_bytes:      Long,   // -1 when unknown (pre-HEAD)
)
```

<Note>
AI Hub pulls on Android **require** an explicit `chipset`. The Rust side only auto-detects on Windows-on-Snapdragon. Use `"SM8750"` for Snapdragon 8 Elite Gen 5.
</Note>

## **LLM data structures**

### `LlmCreateInput`

```kotlin
data class LlmCreateInput(
    val model_name:     String,

    val tokenizer_path: String? = null,
    val config:         ModelConfig,

    val device_id:      String? = null,   // see Device selection
)
```

### `ChatMessage`

```kotlin
data class ChatMessage(
    var role:    String,   // "system" | "user" | "assistant"
    var content: String,
)
```

### `ModelConfig`

```kotlin
data class ModelConfig(
    var nCtx:                  Int     = 2048,     // context size; 0 = model default
    var nThreads:              Int     = 8,
    var nThreadsBatch:         Int     = 8,
    var nBatch:                Int     = 2048,
    var nUBatch:               Int     = 512,
    var nSeqMax:               Int     = 1,
    var nGpuLayers:            Int     = 0,        // see note below
    val chat_template_path:    String  = "",
    val chat_template_content: String  = "",
    val max_tokens:            Int     = 2048,
    val enable_thinking:       Boolean = false,
    val verbose:               Boolean = false,
)
```

<Note>
`nGpuLayers` is rewritten by the JNI when you pass a `device_id` alias: `cpu` forces 0, `hybrid` forces 999. For `gpu` and `npu` the value you pass is used as-is — set it to 999 to offload all layers.
</Note>

### `GenerationConfig` and streaming

The streaming API emits:

```kotlin
sealed class LlmStreamResult {
    data class Token(val text: String)    : LlmStreamResult()
    object         Completed              : LlmStreamResult()
    data class Error(val throwable: Throwable) : LlmStreamResult()
}
```

Construct a `GenerationConfig` with `maxTokens`, sampling params, and grammar as needed. Full fields are in [`GenerationConfigSample`](https://github.com/qcom-ai-hub/geniex/blob/main/examples/android/app/src/main/java/com/geniex/demo/GenerationConfigSample.kt).
