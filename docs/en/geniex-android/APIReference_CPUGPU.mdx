The `"llama_cpp"` plugin runs any GGUF model on CPU, Adreno GPU, or the Hexagon NPU through llama.cpp's per-tensor HTP+CPU scheduler. Device selection is controlled by `device_id`; see [Basics — Device selection](/en/geniex-android/APIReference#device-selection).

All examples below assume the model has already been pulled with `ModelManagerWrapper.pullFlow(...)` — see [Basics — Model manager](/en/geniex-android/APIReference#model-manager). Use the paths returned by `getPaths()` instead of constructing paths yourself.

---

## **LLM**

### Load and generate

```kotlin
// Resolve on-disk paths for a previously-pulled model.
val paths = ModelManagerWrapper.getPaths("unsloth/Qwen3-0.6B-GGUF")
    ?: error("Model not downloaded")

LlmWrapper.builder()
    .llmCreateInput(
        LlmCreateInput(
            model_name = paths.model_name,
            model_path = paths.model_path,
            config     = ModelConfig(nCtx = 4096),
            plugin_id  = "llama_cpp",
            device_id  = null,       // null → plugin default (hybrid on Snapdragon)
        )
    )
    .build()
    .onSuccess { llmWrapper = it }
    .onFailure { println("Error: ${it.message}") }

val chat = arrayListOf(ChatMessage("user", "What is AI?"))

llmWrapper.applyChatTemplate(chat.toTypedArray(), null, false).onSuccess { t ->
    val gen = GenerationConfig(maxTokens = 2048)
    llmWrapper.generateStreamFlow(t.formattedText, gen).collect { result ->
        when (result) {
            is LlmStreamResult.Token     -> print(result.text)
            is LlmStreamResult.Completed -> println("\nDone")
            is LlmStreamResult.Error     -> println("Error: ${result.throwable}")
        }
    }
}
```

### Device variants

| Goal | `device_id` | `nGpuLayers` | Notes |
|---|---|---|---|
| Snapdragon fast path (recommended) | `"hybrid"` or `null` | forced to 999 by JNI | Per-tensor HTP+CPU scheduler — fastest on Snapdragon. |
| Pure CPU | `"cpu"` | forced to 0 | Works on any ARM64 device. |
| Adreno GPU (OpenCL) | `"gpu"` | ≥ 1 (use 999 to offload all) | Requires Adreno OpenCL support. |
| Pinned single-HTP session | `"npu"` | caller-set (use 999 for all) | Deterministic layout; slower than `hybrid` on LLMs. |

```kotlin
// Example: explicit Adreno GPU.
LlmWrapper.builder().llmCreateInput(
    LlmCreateInput(
        model_name = paths.model_name,
        model_path = paths.model_path,
        config     = ModelConfig(nCtx = 4096, nGpuLayers = 999),
        plugin_id  = "llama_cpp",
        device_id  = "gpu",
    )
).build()
```

<Note>
Some model families are incompatible with `hybrid` and are pinned to `npu` automatically by the native SDK. `gpt-oss-*` is the current case. No caller-side branching is needed — pass `null` and the right thing happens.
</Note>

## **VLM**

GGUF vision-language models need two artifacts: the LLM weights (`model_path`) and the vision projection (`mmproj_path`). Both come back from `getPaths()`:

```kotlin
val paths = ModelManagerWrapper.getPaths("unsloth/Qwen3-VL-2B-Instruct-GGUF")
    ?: error("Model not downloaded")

VlmWrapper.builder()
    .vlmCreateInput(
        VlmCreateInput(
            model_name  = paths.model_name,
            model_path  = paths.model_path,
            mmproj_path = paths.mmproj_path,       // vision projection weights
            config      = ModelConfig(nCtx = 4096),
            plugin_id   = "llama_cpp",
            device_id   = null,                    // hybrid by default
        )
    )
    .build()
    .onSuccess { vlmWrapper = it }

val msg = VlmChatMessage(
    role     = "user",
    contents = listOf(
        VlmContent("image", "/storage/emulated/0/Pictures/example.jpg"),
        VlmContent("text",  "Describe this image."),
    ),
)
val chat = arrayListOf(msg)

vlmWrapper.applyChatTemplate(chat.toTypedArray(), null, false).onSuccess { t ->
    val base = GenerationConfig(maxTokens = 2048)
    // injectMediaPathsToConfig forwards image/audio paths from the chat
    // into the config the native generator consumes.
    val gen = vlmWrapper.injectMediaPathsToConfig(chat.toTypedArray(), base)
    vlmWrapper.generateStreamFlow(t.formattedText, gen).collect { result ->
        when (result) {
            is LlmStreamResult.Token     -> print(result.text)
            is LlmStreamResult.Completed -> println("\nDone")
            is LlmStreamResult.Error     -> println("Error: ${result.throwable}")
        }
    }
}
```

<Note>
Always pass `t.formattedText` (the chat-templated prompt) into `generateStreamFlow`, **not** the raw user text. The native pipeline treats the prompt as already-templated — passing raw text puts the model into completion mode and produces self-repeat / empty-response artifacts on structured input.
</Note>
