There are two ways to run a model on Qualcomm Hexagon NPU:

1. **Qualcomm AI Hub pre-compiled models via `"qairt"`** — purpose-built for specific chipsets (e.g. `SM8750` = Snapdragon 8 Elite Gen 5). Fastest and most memory-efficient path, but limited to the models Qualcomm publishes on AI Hub.
2. **Any GGUF model via `"llama_cpp"`** with `device_id = "npu"` (pinned) or `"hybrid"` (fast path). Works with any community GGUF; see [CPU / GPU / Hybrid](/en/geniex-android/APIReference_CPUGPU).

This page covers path #1. For path #2 see the CPU / GPU / Hybrid doc — the wrapper code is identical; only `plugin_id` changes.

---

## **Supported AI Hub models**

The demo app ships with the following QAIRT models. All are pinned to chipset `SM8750` (Snapdragon 8 Elite Gen 5 / Hexagon v81).

| Modality | Hub repo |
|---|---|
| LLM | `qualcomm/Qwen3-4B-Instruct-2507` |
| VLM | `qualcomm/Qwen2.5-VL-7B-Instruct` |

Beyond these, any model under [Qualcomm AI Hub](https://aihub.qualcomm.com/) that ships a pre-compiled NPU bundle for your chipset can be pulled the same way — register the AI Hub model id in the Rust model manager's QAIRT LLM / VLM registry, and it becomes available to the SDK.

---

## **Download an AI Hub model**

```kotlin
ModelManagerWrapper.pullFlow(
    ModelPullInput(
        model_name = "qualcomm/Qwen2.5-VL-7B-Instruct",
        hub        = HubSource.AUTO,     // AUTO routes `qualcomm/*` to AI Hub
        chipset    = "SM8750",           // REQUIRED on Android
    )
).collect { event ->
    when (event) {
        is ModelManagerWrapper.PullEvent.Progress  -> updateProgressBar(event.files)
        ModelManagerWrapper.PullEvent.Completed    -> println("done")
        is ModelManagerWrapper.PullEvent.Error     -> println("err ${event.code}: ${event.message}")
    }
}
```

<Note>
On Android you **must** pass `chipset` for AI Hub pulls. The Rust side only auto-detects on Windows-on-Snapdragon. Omitting it returns `GENIEX_ERROR_COMMON_INVALID_INPUT`.
</Note>

## **LLM**

```kotlin
val paths = ModelManagerWrapper.getPaths("qualcomm/Qwen3-4B-Instruct-2507")
    ?: error("Model not downloaded")

LlmWrapper.builder()
    .llmCreateInput(
        LlmCreateInput(
            model_name = paths.model_name,
            model_path = paths.model_path,
            config     = ModelConfig(max_tokens = 2048, enable_thinking = false),
            plugin_id  = "qairt",        // QAIRT is NPU-only
            device_id  = null,           // null → NPU (plugin default)
        )
    )
    .build()
    .onSuccess { llmWrapper = it }
    .onFailure { println("Error: ${it.message}") }

val chat = arrayListOf(ChatMessage("user", "What is AI?"))

llmWrapper.applyChatTemplate(chat.toTypedArray(), null, false).onSuccess { t ->
    llmWrapper.generateStreamFlow(t.formattedText, GenerationConfig()).collect { result ->
        when (result) {
            is LlmStreamResult.Token     -> print(result.text)
            is LlmStreamResult.Completed -> println("\nDone")
            is LlmStreamResult.Error     -> println("Error: ${result.throwable}")
        }
    }
}
```

<Note>
QAIRT rejects `nGpuLayers != 0` and `nCtx != 0` with `PARAM_NOT_SUPPORTED` — the KV cache and context length are fixed at compile time by the AI Hub bundle. Leave both at their defaults (the SDK handles this for you) and use `max_tokens` / `enable_thinking` only.
</Note>

## **VLM**

```kotlin
val paths = ModelManagerWrapper.getPaths("qualcomm/Qwen2.5-VL-7B-Instruct")
    ?: error("Model not downloaded")

VlmWrapper.builder()
    .vlmCreateInput(
        VlmCreateInput(
            model_name  = paths.model_name,
            model_path  = paths.model_path,
            mmproj_path = paths.mmproj_path,   // vision encoder .bin (populated for AI Hub VLMs)
            config      = ModelConfig(max_tokens = 2048, enable_thinking = false),
            plugin_id   = "qairt",
            device_id   = null,
        )
    )
    .build()
    .onSuccess { vlmWrapper = it }

val msg = VlmChatMessage(
    role     = "user",
    contents = listOf(
        VlmContent("image", "/storage/emulated/0/Pictures/cat.jpg"),
        VlmContent("text",  "What's in this image?"),
    ),
)
val chat = arrayListOf(msg)

vlmWrapper.applyChatTemplate(chat.toTypedArray(), null, false).onSuccess { t ->
    val base = GenerationConfig(maxTokens = 2048)
    val gen  = vlmWrapper.injectMediaPathsToConfig(chat.toTypedArray(), base)
    vlmWrapper.generateStreamFlow(t.formattedText, gen).collect { result ->
        when (result) {
            is LlmStreamResult.Token     -> print(result.text)
            is LlmStreamResult.Completed -> println("\nDone")
            is LlmStreamResult.Error     -> println("Error: ${result.throwable}")
        }
    }
}
```

<Note>
Pass the **chat-templated** prompt (`t.formattedText`) to `generateStreamFlow`, never the raw user text. The QAIRT VLM pipeline treats its prompt argument as already-templated; sending raw text puts the model into completion mode and produces degenerate output on structured inputs.
</Note>

## **Authoritative `plugin_id` from paths**

If your UI lets the user pick CPU / GPU / NPU but you want to honour what the model was actually compiled for, use `paths.plugin_id`:

```kotlin
val paths    = ModelManagerWrapper.getPaths(name) ?: return
val pluginId = paths.plugin_id          // "qairt" for AI Hub models, "llama_cpp" for GGUF
```

This is what the demo app does — the plugin picker dialog is used only to pick `device_id` for `llama_cpp` models.
