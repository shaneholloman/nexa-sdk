## **Overview**

GenieX includes a built-in local inference server that exposes an **OpenAI-compatible API**. 
This lets you run models on-device and connect them to applications or frameworks that fit the OpenAI protocol 
— including agentic workflows like **LangChain**, AI-native apps like **OpenClaw**, and your own custom applications. 
No cloud dependency — just a local endpoint backed by Qualcomm hardware acceleration.

## **Getting Started**

If you haven't already, pull a model before making requests. For example, pull Qwen3:

```bash bash
geniex pull qualcomm/Qwen3-4B-Instruct-2507
```

To use the API, first open a terminal from the project root. Then, start the GenieX server by:

```bash bash
geniex serve
```

The server runs on `http://127.0.0.1:18181` by default. \
Keep the terminal that runs the server open, and make your requests from another terminal tab. \
To see a full list of configurable options for the server, run `geniex serve -h`


## **/v1/chat/completions**

Creates a model response for a given conversation. Supports LLM(text-only) and VLM (image+text).

### **Use LLM**

#### **Request body**

```json Example Value
{
  "model": "qualcomm/Qwen3-4B-Instruct-2507",
  "messages": [
    {"role": "user", "content": "Hello! Briefly introduce yourself."}
  ],
  "max_tokens": 256,
  "temperature": 0.7,
  "stream": false
}
```

#### **Usage Example**

Open `http://127.0.0.1:18181` in your browser to access the built-in Swagger UI.

**Step 1.** Expand the `POST /v1/chat/completions` endpoint to view the example request body and schema.

### **Use VLM**

#### **Request body**

`image_url` can be one of the following:

- A local file path on the server machine (requires server access to the file), e.g., `C:/Users/Username/Pictures/photo.jpg`. The `file://` prefix is optional.

First, download a sample image to the current directory, add below request to your 

```json Example Value
{
  "model": "qualcomm/Qwen2.5-VL-7B-Instruct",
  "messages": [
    {
      "role": "user",
      "content": [
        {"type": "text", "text": "Describe this image succinctly."},
        {"type": "image_url", "image_url": {"url": "</path/to/image>"}}
      ]
    }
  ]
}
```

#### **Usage Example**

Open `http://127.0.0.1:18181` in your browser to access the built-in Swagger UI.

**Step 1.** Expand the `POST /v1/chat/completions` endpoint to view the example request body and schema.

![Swagger UI showing the chat completions endpoint with example request body](/Images/server/curl-1.png)

**Step 2.** Click **Try it out**, replace the request body with a VLM payload (point `image_url.url` to a local image path), then click **Execute**.

![Editing the VLM request body in Try-it-out mode before executing](/Images/server/curl-4.png)

**Step 3.** View the response — a `200` status with the model's description of the image.

![Response body showing a successful 200 response with the VLM's image description](/Images/server/curl-5.png)


## **Python Example (OpenAI SDK)**

Because the server speaks the OpenAI protocol, you can point the official `openai` Python client at the local endpoint and reuse any existing OpenAI code. Install the SDK with `pip install openai`, then:

```python python
from openai import OpenAI

client = OpenAI(
    base_url="http://127.0.0.1:18181/v1",
    api_key="geniex",  # any non-empty string; the server does not check it
)

stream = client.chat.completions.create(
    model="unsloth/Qwen3-4B-GGUF:Q4_0",  # org/repo[:quant] — Q4_0 has the best Hexagon NPU support
    messages=[
        {"role": "user", "content": "Hello! Briefly introduce yourself."},
    ],
    max_tokens=256,
    temperature=0.7,
    stream=True,
)

for chunk in stream:
    delta = chunk.choices[0].delta.content
    if delta:
        print(delta, end="", flush=True)
print()
```

<Note>
Replace the `model` value with a model you have already pulled. `geniex serve` does not auto-download models. The optional `:<quant>` suffix selects a quantization variant (e.g. `Q4_0`, `Q4_K_M`, `Q8_0`) — `Q4_0` is recommended for the llama.cpp Hexagon NPU backend.
</Note>
