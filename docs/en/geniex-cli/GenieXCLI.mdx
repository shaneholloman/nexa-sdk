## **geniex pull**

Download a model and store it locally.
After entering the pull command, you will be guided through a setup process to choose the model type, main model file, tokenizer (optional), and extra files (optional).

### **General Behavior**

After running `geniex pull <model-name>`, the CLI will prompt:

1. **Quant version selection**\
   If the model supports multiple quantized versions, you will see a menu like this:

   ```bash Quant version selection
   Choose a quant version to download
   > Q4_K_M     [1.2 GiB] (default)
     Q8_0       [2.0 GiB]
     F16        [3.8 GiB]
   ```

   Select a quant version you prefer.

2. **Download begins**\
   After selection, the model files will start downloading automatically.

### **LLM**

```bash bash
geniex pull Qualcomm/Qwen3-4B-Instruct
```

### **VLM**

```bash bash
geniex pull NexaAI/Qwen2.5-Omni-3B-GGUF
```

## **geniex list**

Display all downloaded models in a table with their names and sizes.

```bash bash
geniex list
```

## **geniex remove**

Remove a specific local model by name.

```bash bash
geniex remove unsloth/Qwen3-0.6B-GGUF
```

## **geniex clean**

Delete all locally cached models.

```bash bash
geniex clean
```

## **geniex infer**

Run inference with a specified model. The model must be downloaded and cached locally.

### **Helper menu**

```bash bash
geniex infer -h
```

Show help menu for `geniex infer`.

### **LLM**

Launch an interactive chat session with the language model.

```bash bash
geniex infer unsloth/Qwen3-0.6B-GGUF
```

Use the `--think` option to control whether the model outputs its internal reasoning process.
- `--think=false` : The model responds directly without showing reasoning.
- `--think=true` : The model displays its reasoning steps before the final response.

Example with reasoning enabled:
```bash bash
geniex infer Qualcomm/Qwen3-4B-Instruct --think=true
```

### **VLM**

Text-only or response from image file (interactive image input):

```bash bash
geniex infer NexaAI/Qwen2.5-Omni-3B-GGUF
```

If you only want text input, simply launch the command and begin chatting.\
If you'd like the model to response from an image, provide the **absolute path** to an image at the end of your message.\
Example prompt: `Describe this picture </path/to/image.png>`

## **geniex serve**

Launch the GenieX inference server using REST API.

### **Helper menu**

```bash bash
geniex serve -h
```

Show help menu for `geniex serve`.

### **Start serve**

For example, start a local inference server bound to 127.0.0.1:8080. The server supports OpenAI-compatible APIs.

```bash bash
geniex serve --host 127.0.0.1:8080
```

You can test it via:

<CodeGroup>
    ```bash Windows (cmd)
    curl -X POST http://127.0.0.1:8080/v1/chat/completions -H "Content-Type: application/json" -d "{\"model\": \"Qualcomm/Qwen3-4B-Instruct\", \"messages\": [{\"role\": \"user\", \"content\": \"Hello!\"}], \"max_tokens\": 64}"
    ```

    ```bash Linux
    curl -X POST http://127.0.0.1:8080/v1/chat/completions \
    -H "Content-Type: application/json" \
    -d '{
      "model": "Qualcomm/Qwen3-4B-Instruct",
      "messages": [{"role":"user","content":"Hello!"}],
      "max_tokens": 64
    }'
    ```

</CodeGroup>

This will send a POST request to the local /v1/completions endpoint to prompt the Qualcomm/Qwen3-4B-Instruct model with "What is the capital of France?" and return a response with up to 100 tokens.
