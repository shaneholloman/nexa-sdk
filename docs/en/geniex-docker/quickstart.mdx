## **Installation**

### **Pull Docker Image**

login to GitHub Container Registry (GHCR) using your GitHub PAT (Personal Access Token):

```bash bash
export GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxxxx
docker login ghcr.io -u zackli_QCOM -p "$GITHUB_TOKEN"
```

Pull the latest GenieXSDK Docker image from Docker Hub:

```bash bash
docker pull ghcr.io/qcom-ai-hub/geniex-cli:latest
```

## **Usage Modes**

GenieXSDK Docker supports two usage modes: server mode for REST API access and interactive CLI mode for direct model inference.


### **CLI Mode (Interactive)**

Run GenieXSDK in interactive CLI mode for direct model inference:

```bash bash
docker run -it --rm --privileged \
  -v "$PWD/data:/data" \
  -v /usr/lib:/opt/qcom-lib:ro \
  ghcr.io/qcom-ai-hub/geniex-cli:latest \
  infer Qwen/Qwen3-0.6B-GGUF
```

<Note>
The `-it` flags enable interactive mode. Replace `Qwen/Qwen3-0.6B-GGUF` with any supported model name. For a complete list of supported models, see the [Overview](/en/geniex-docker/overview#supported-models) page.
</Note>


### **Server Mode (Detached)**

Run GenieXSDK in server mode to expose a REST API endpoint. This mode runs in the background:

```bash bash
docker run -it --rm --privileged \
  -v "$PWD/data:/data" \
  -v /usr/lib:/opt/qcom-lib:ro \
  --network=host \
  ghcr.io/qcom-ai-hub/geniex-cli:latest \
  serve
```

<Note>
The `--privileged` flag is required for NPU access. Replace `YOUR_LONG_TOKEN_HERE` with your actual GenieX token. You can obtain a token by creating an account at [sdk.geniex.ai](https://sdk.geniex.ai) and generating one in **Deployment → Create Token**.
</Note>

`geniex serve` won't auto download models. Make sure to pre-download the models you intend to use.

The server will be accessible at `http://localhost:18181`. For detailed API documentation, see the [REST API](/en/geniex-docker/RestAPI) page.
