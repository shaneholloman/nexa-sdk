## **Prerequisites**

- The CLI installed — see [Install](/en/run/cli/install).
- Familiarity with [backend choice](/en/get-started/choose-backend) — `qairt` for AI Hub bundles, `llama_cpp` for any GGUF.

## **Run your first model**

<Tabs>
  <Tab title="Windows ARM64">
    ### **QAIRT plugin (AI Hub)**

    **Language model:**

    ```powershell windows
    geniex infer qualcomm/Qwen3-4B
    ```

    **Multimodal model:**

    ```powershell windows
    geniex infer qualcomm/Qwen2.5-VL-7B-Instruct
    ```

    ### **llama.cpp plugin (GGUF)**

    Pick `Q4_0` when prompted — it has the best Hexagon NPU support.

    **Language model:**

    ```powershell windows
    geniex infer unsloth/Qwen3.5-0.8B-GGUF
    ```

    **Multimodal model:**

    ```powershell windows
    geniex infer Qwen/Qwen3-VL-2B-Instruct-GGUF
    ```

    When prompted:
    - **Model type** — `vlm` for vision-language models, `llm` for text-only models. For `Qwen3.5` and `Gemma4`, pick `llm` for now (multimodal support coming soon).
    - **Quantization** — `Q4_0` for best Hexagon NPU performance.

    <Tip>To try other GGUF models, copy any compatible GGUF path from Hugging Face and substitute it into the command above. See [Bring your own model](/en/models/bring-your-own).</Tip>
  </Tab>

  <Tab title="Linux ARM64">
    With the [image pulled](/en/run/cli/install), run a model interactively:

    ```bash bash
    docker run -it --rm --privileged \
      -v "$PWD/data:/data" \
      -v /usr/lib:/opt/qcom-lib:ro \
      ghcr.io/qcom-ai-hub/geniex-cli:latest \
      infer Qwen/Qwen3-0.6B-GGUF
    ```

    For an HTTP API, see [Local server](/en/run/cli/local-server).
  </Tab>
</Tabs>
