The Linux Docker path runs through a container image built for Linux ARM64 with NPU access (Dragonwing IQ9 and similar IoT platforms).

### **Prerequisites**

- Linux ARM64 host with Docker installed.
- Qualcomm IoT chipset — see [Supported platforms](/en/get-started/platforms).
- GitHub Personal Access Token (PAT) with read access to GHCR.

### **Pull the image**

Authenticate to GitHub Container Registry, then pull:

```bash bash
export GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxxxx
docker login ghcr.io -u <your-github-username> -p "$GITHUB_TOKEN"

docker pull ghcr.io/qcom-ai-hub/geniex-cli:latest
```

<Note>
  If `docker login` fails or `docker pull` returns `permission denied ... /var/run/docker.sock`, see [Troubleshooting → Linux](/en/resources/troubleshooting#linux) for the PAT scope and `docker` group fixes.
</Note>

### **Run interactively**

Drop into an interactive shell inside the container:

```bash bash
docker run -it --rm --privileged \
  -v "$PWD/data:/data" \
  -v /usr/lib:/opt/qcom-lib:ro \
  ghcr.io/qcom-ai-hub/geniex-cli:latest
```

### **Verify**

Then run a test command from the container shell:

```bash bash
geniex --help
```

<Note>The `--privileged` flag is required for NPU access.</Note>

For server mode, see [Local server](/en/run/cli/local-server).
